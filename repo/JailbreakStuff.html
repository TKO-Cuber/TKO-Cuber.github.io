SwitchBoard

This section describes everything I know about Apple's Internal Prototype firmware, SwitchBoard.






The Filesystem

I'll have to admit, the filesystem is fun to poke around

Looking at the filesystem itself, SwitchBoard resembles iOS very much. They both have the standard basic filesystem structure, containing /var, /System, /Library, etc.

SwitchBoard does have another folder that stock iOS doesn't have, called AppleInternal, which holds the internal apps like SwitchBoard.app (more on that later) and Signage.app and stuff.

The Fun Stuff

Getting it up and running

Got a copy? Cool. Let's get it running!

First of all, let's check if you can install it, safely that is.

If you have a copy meant for the 3G and you don't like an exploding iPhone 3G, don't install it!

The 3G switchboard (the popular one) has a few very big issues. The icons disappear, and battery expansion is possible, and in rare cases, it can break NVRAM- yikes!

Now, as long as you have a full IPSW (this part only covers the 3GS BTW- maybe i4 later idk... you can install any IPSW with kloader, but that's for another day), you're good to go.




Step One

Pwning DFU for the restore

Open Sn0wBreeze, and click ok until you get to the main screen. In the top left corner, click "run iREB". Click no on the popup (is there an ATV2 SwitchBoard?) then follow the instructions to enter pwned dfu mode. Easy enough.

Step Two

Restoring

Open iTunes. A popup should come up saying it found a phone in recovery mode. good. Shift+Click restore, then select the switchboard ipsw. Let it restore. If there are errors, check theiphonewiki. They have a ton of help.

Step Three

Install the Alloc8 exploit

You'll need linux for this. There are guides online for installing linux off of a USB, check those out. Once you are in linux, download XPWNDFU (search it up- it's on GitHub). Extract the zip, open a terminal in the XPWNDFU folder containing the file called "setup". Run ./setup for it to set up.

Now cd into the ipwndfu folder and run sudo ./ipwndfu -p te enter pwned DFU mode. If it fails, unplug the phone and try again. Now run sudo ./ipwndfu -x to install the exploit to NOR, and wait for SwitchBoard to load up!
